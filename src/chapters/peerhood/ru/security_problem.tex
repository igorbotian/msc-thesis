\Paragraph{Описание проблемы безопасности в целом}
\Sentence
В наше время проблема безопасности информации стоит чрезвычайно остро.
\Sentence
Особенно актуально это звучит в контексте \RussianAbbreviation{ПДУ}, когда личные данные 
пользователя задействованы в процессе коммуникации.
\Sentence
Поэтому необходимо учитывать риски безопасности, связанные с нарушением \Emphasis{целостности}, 
\Emphasis{конфиденциальности} и \Emphasis{доступности} этих данных.

\Paragraph{Актуальность вопроса безопасности для PeerHood}
\Sentence
В разработке проекта \PeerHood направление, связанное с обеспечением безопасности, является одним 
из наиболее приоритетных \Reference{PeerHoodSpecification}. 
\Sentence
Основными причинами этого являются следующие:
\begin{enumerate}
	\item PeerHood имеет полный доступ к личной информации, хранящейся на устройстве пользователя.
	\item В процессе коммуникации PeerHood передаёт данные пользователя, в том числе личные.
	\item PeerHood является сетевым приложением, поэтому безопасности передаваемых им данных также 
		зависит от их безопасности сетевой технологии, которую он использует в конкретный момент 
		времени.
	\item Безопасность личных данных пользователя также зависит от безопасности самого ПО PeerHood.
\end{enumerate}

\Paragraph{Необходимость проведения анализа проекта}
\Sentence
Создание механизма безопасности для \PeerHood является одной из наиболее приоритетных задач 
\Reference{PeerHoodSpecification}.
\Sentence
Первым шагом на пути к её реализации является анализ проекта на безопасность для выяснения того, 
какие существуют угрозы безопасности, какие места в проекте являются слабыми и т.д..
\Sentence
Этому вопросу посвящён следующий раздел данной диссертации.
